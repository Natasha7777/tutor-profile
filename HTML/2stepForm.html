Урок 2

С этого момента я не буду приводить код нашего документа полностью. Однако, вы должны понимать, что приводимый далее код вы должны вставлять между <FORM> и </FORM>.

Итак, наиболее распространенный тип (TYPE) поля <INPUT> это текстовой (TEXT).

     
<INPUT TYPE=TEXT>

Вот такое оно поле для ввода текста (одной строки текста).


Вся информация, которую мы вводим, должна иметь имя (NAME).

     
<INPUT TYPE=TEXT NAME="ADDRESS">

Визуально это никак не отображается.


Когда ваш посетитель напечатает в этом поле свой адрес (например, 1313 Улица Пересмешника), то он становится выводимым значением, которое будет связано с ADDRESS (адресом), и в результате после расшифровки данных почтовой программой или расшифровщиком данных формы вы получите: "АДРЕС (ADDRESS)=1313 Улица Пересмешника". Мы можем сами ввести значение (VALUE) для поля формы:

     
<INPUT TYPE=TEXT NAME="ADDRESS" VALUE="44 Cherry St">


Это значит, что ADDRESS будет связан со значением "44 Cherry St", пока пользователь (ваш посетитель) не изменить его на свое.

Кстати, кавычки ставьте обязательно, не пропускайте их, не забывайте, чтобы форма корректно работала и отображалась.

Мы также можем задать свой размер для текстового поля:

     
<INPUT TYPE=TEXT NAME="ADDRESS" VALUE="44 Cherry St" SIZE="10">


<INPUT TYPE=TEXT NAME="ADDRESS" VALUE="44 Cherry St" SIZE="20">


<INPUT TYPE=TEXT NAME="ADDRESS" VALUE="44 Cherry St" SIZE="30">


По умолчанию размер текстового поля равен 20-ти. По умолчанию - это значит, что броузер так отобразит какой-то элемент, даже если вы не задали специфических значений для него (т.е. если вы не напишите SIZE="20" для поля формы, то его размер все равно будет 20-ть, т.к. он такой по умолчанию).

Давайте теперь удалим VALUE="44 Cherry St". И введем размер для поля равный 30-ти.

     
<INPUT TYPE=TEXT NAME="ADDRESS" SIZE="30">


Если мы хотим, то мы можем задать максимальное количество символов, которое ваш посетитель может ввести в текстовое поле.

     
<INPUT TYPE=TEXT NAME="ADDRESS" SIZE="30" MAXLENGTH="10">

Попробуйте напечатать в этом поле больше десяти символов - у вас ничего не получится! :)


Зачем нам нужна такая возможность ограничивать нашего посетителя? А представьте, что вдруг кому-то взбредет хулиганства ради послать вам текст всей Библии (примечание переводчика: или всю "Войну и Мир" Толстого :). То-то вы удивитесь, когда будете час-другой почту скачивать.

Очень схоже с текстовым полем (TYPE=TEXT) поле для ввода пароля (TYPE=PASSWORD). Оно фактически такое же, только не отображает вводимые вами символы, а маркирует их звездочками (***). Броузер перешлет вам то, что вводил пользователь (ввиде нормальных символов), просто окружающим не будет видно, что вводил ваш посетитель в поле формы (примечание переводчика: весьма полезная вещь, если ваш посетитель сидит в интернет-кафе или офисе, т.е. местах, где посторонний человек может подсмотреть вводимую информацию).

     
<INPUT TYPE=PASSWORD>

Попробуйте напечатать в этом поле что-нибудь.


Итак, вспомним, что каждое поле (INPUT) должно иметь имя (NAME):

     
<INPUT TYPE=PASSWORD NAME="USER PASSWORD">


SIZE, VALUE, и MAXLENGTH - эти атрибуты также работают с полем для ввода пароля, как и с текстовым полем.

Кстати, надеюсь вы знаете, что теги (<INPUT>) говорят броузеру что-то сделать. А атрибуты тегов (TYPE, NAME, MAXLENGTH и т.д.) говорят броузеру, как это сделать.

Далее мы с вами научимся создавать "переключатели" (Radio Buttons) и "флажки" (Check Boxes). Переключатели позволяют пользователю выбрать одну из нескольких опций (параметров). Флажки позволяют выбрать пользователю одну, несколько или все опции.

     
<INPUT TYPE=RADIO NAME="BEST FRIEND">

Это переключатель.


Добавим еще парочку переключателей.

     
<INPUT TYPE=RADIO NAME="BEST FRIEND">
<INPUT TYPE=RADIO NAME="BEST FRIEND">
<INPUT TYPE=RADIO NAME="BEST FRIEND">

Можно по ним пощелкать. Больше одного выбрать не получится.


Хм... Думаю, что лучше их разнести по разным строкам:

     
<INPUT TYPE=RADIO NAME="BEST FRIEND"><BR>
<INPUT TYPE=RADIO NAME="BEST FRIEND"><BR>
<INPUT TYPE=RADIO NAME="BEST FRIEND"><P>


Заметьте, все переключатели имеют одно и тоже имя (NAME). Почему? Это вам станет ясно чуть позднее.

Каждому переключателю должно быть задано свое собственное значение

     
<INPUT TYPE=RADIO NAME="BEST FRIEND" VALUE="Ed"><BR>
<INPUT TYPE=RADIO NAME="BEST FRIEND" VALUE="Rick"><BR>
<INPUT TYPE=RADIO NAME="BEST FRIEND" VALUE="Tom"><P>


Теперь зададим описание каждому переключателю:

     
<INPUT TYPE=RADIO NAME="BEST FRIEND" VALUE="Ed"> Эд Холлеран<BR>
<INPUT TYPE=RADIO NAME="BEST FRIEND" VALUE="Rick"> Рик Вайнберг<BR>
<INPUT TYPE=RADIO NAME="BEST FRIEND" VALUE="Tom"> Том Студд<P>

Эд Холлеран
Рик Вайнберг
Том Студд


Вы можете видоизменить описания любыми знакомыми вам html-тегами (прим. переводчика: понятно, что html-тегами, которые мы применяем к тексту, т.е. изменить цвет описаний, размер, шрифт и т.д.).

Вот фактически наша форма с опросом (где мы используем переключатели) и доделана. Чтобы было понятнее пользователю: мы можем приписать какой-либо вопрос к форме (например, мы хотим опросить знакомых, кто их лучший друг из этих трех молодых людей). Кроме того при помощи атрибута CHECKED мы можем выбрать, какая опция будет выбрана по умолчанию (пока пользователь не выберет ту опцию, которую считает нужной).

     
Кто ваш лучший друг?<BR>
<INPUT TYPE=RADIO NAME="BEST FRIEND" VALUE="Ed" CHECKED> Эд Холлеран<BR>
<INPUT TYPE=RADIO NAME="BEST FRIEND" VALUE="Rick"> Рик Вайнберг<BR>
<INPUT TYPE=RADIO NAME="BEST FRIEND" VALUE="Tom"> Том Студд<P>

Кто ваш лучший друг?

Рик Вайнберг
Том Студд


Еще раз повторюсь, пользователь при данном типе полей формы (переключателей) может выбрать только одну опцию. Выбор вашего посетителя будет переслан вам как: BEST FRIEND=Ed (или другой молодый человек, смотря кого выберут).

Теперь мы с вами будем учиться создавать форму с "флажками". Создание такой формы аналогично созданию формы с "переключателями".

     
<INPUT TYPE=CHECKBOX NAME="Ed">

Щелкните, чтобы выбрать флажок, еще раз щелкните - чтобы убрать выделение.


Добавим еще несколько флажков. Каждому зададим свое имя (NAME). Если хотите, то вы можете разнести флажки по разным строкам.

     
<INPUT TYPE=CHECKBOX NAME="Ed"><BR>
<INPUT TYPE=CHECKBOX NAME="Rick"><BR>
<INPUT TYPE=CHECKBOX NAME="Tom"><BR>
<INPUT TYPE=CHECKBOX NAME="BM"><P>




Пощелкайте по флажкам, вы можете выбрать сразу несколько из них.


Всем флажкам должно быть задано одинаковое значение:

     
<INPUT TYPE=CHECKBOX NAME="Ed" VALUE="YES"><BR>
<INPUT TYPE=CHECKBOX NAME="Rick" VALUE="YES"><BR>
<INPUT TYPE=CHECKBOX NAME="Tom" VALUE="YES"><BR>
<INPUT TYPE=CHECKBOX NAME="BM" VALUE="YES"><P>


Итак заметьте для "флажков" имя (NAME) разное, а значение (VALUE) для всех одинаковое, а для "переключателей" наооборот (NAME - одинаковое, а VALUE разное). Не переживайте, когда я долгими ночами изучал спецификацию по html, меня тоже этот момент приводил в замешательство, но потом в голове все разложилось по полочкам, и теперь я умный :).
